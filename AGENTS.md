# PI-CATPPUCCIN-TUI KNOWLEDGE BASE

**Last updated:** 2026-06-03
**Branch:** main

## OVERVIEW

A **Pi package** (not an application) that ships exact Catppuccin themes for the
Pi coding-agent TUI, plus an optional, off-by-default extension that adds
Catppuccin-flavored TUI polish. Two deliverables:

1. **Themes** — four exact Catppuccin flavors (Latte, Frappé, Macchiato, Mocha)
   as Pi theme JSON. Macchiato is the signature flavor.
2. **Extension** — a `/catppuccin-tui` command that toggles three optional
   enhancements (streaming indicator, footer status entry, compact footer).
   All off by default; installing the package never changes Pi behavior until a
   user opts in.

Distributed via npm and meant to be discoverable on https://pi.dev/packages.

## HOW THEMES ARE BUILT / VALIDATED

- **Build:** `npm run build:themes` runs `scripts/generate-themes.mjs`, which writes
  all four `themes/*.json` from the in-script `palette` and `colorTokens`. Themes are
  derived artifacts; the script is the single source of truth.
- **Validate:** `npm run validate` runs `scripts/validate-themes.mjs`, asserting each
  theme defines exactly the 51 required Pi color tokens, has one of the 4 exact theme
  names, uses 6-digit hex for every `vars` value, and references only known variables.
- **JSON shape:** `npm pack --dry-run --json` previews publish contents against the
  `files` allowlist in `package.json`.

## CONVENTIONS

- **Themes are generated, not hand-edited.** Treat `themes/*.json` like build output
  (analogous to a generated lockfile). Source of truth is `generate-themes.mjs`.
- **Token contract is enforced.** Generate and validate must agree: exactly 51 tokens,
  4 exact theme names. Change one side → change the other, then re-validate.
- **Enhancements stay opt-in.** The extension ships all features off; never make an
  enhancement default-on (see SESSION_HANDOFF risk boundaries).
- **Palette values are copied from official Catppuccin**, not invented. Source:
  https://catppuccin.com/palette/ and the Catppuccin style guide. Don't eyeball colors.
- **Evidence before "done".** Run `npm run validate` (and `npm pack --dry-run` for
  publish-shape changes); paste the result into `feature_list.json`.

## NOTES

- `node` (ES modules, top-level await) and `jq` are the only build/validate tooling.
  No bundler, no test framework in this repo.
- Peer deps `@earendil-works/pi-coding-agent` and `@earendil-works/pi-tui` are provided
  by the host Pi install; the extension imports types from them.
